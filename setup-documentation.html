Domain & Subdomain Mapping

For a professional production environment, use dedicated subdomains for each service.

Admin API admin-api.yourdomain.com
Client API api.yourdomain.com
Admin Dashboard admin.yourdomain.com

Nginx Reverse Proxy

Configure Nginx to route traffic to the PM2 processes and serve static assets.

/etc/nginx/sites-available/shuttle 
# Admin API Subdomain
server {
    server_name admin-api.yourdomain.com;
    location / {
        proxy_pass http://localhost:3002;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

# Client API Subdomain
server {
    server_name api.yourdomain.com;
    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

Change Default Country & Currency

The defaults for the application are seeded from the API settings. Change the values in apps/admin-api/src/api/seeders/settings.json.

settings.json (Subset) 
{
  "general": {
    "default_country": "IN",    // e.g., "US", "GB"
    "default_currency": "₹",    // e.g., "$", "£"
    "time_format": "h:mm A",
    "timezone": "Asia/Kolkata"
    // ...
  }
}

Change Enabled Payment Gateway

You can set the default enabled payment gateway before seeding by editing apps/admin-api/src/api/seeders/payment_gateways.json. Change is_enabled value to 1 for your preferred gateway and 0 for others.

payment_gateways.json 
[
  {
    "name": "is_enabled",
    "site": "Razorpay",
    "value": "1"   // <--- Indicates Razorpay is enabled
  },
  {
    "name": "is_enabled",
    "site": "Paystack",
    "value": "0"   // <--- Indicates Paystack is disabled
  }
]

01 Shuttle Android Apps – Installation Guide

This guide explains how to install and configure the Shuttle User App and Shuttle Driver App Android projects.

Both applications require configuration such as Firebase setup, Google Maps API key, backend base URL, and app branding before running the project.

Note: OTP authentication is NOT handled by Firebase.

The OTP system is managed by a third-party OTP service through the backend API.

02 1. System Requirements

Before starting the installation, make sure the following tools are installed.

Required Software

  • Android Studio (Latest Version recommended)
  • JDK 11 or higher
  • Gradle (comes with Android Studio)
  • Firebase Account
  • Google Cloud Console Account

    • Recommended Testing Device

  • Android device or emulator running **Android 7.0 or higher**

    • 03 2. Project Structure

    The project contains two Android applications.

    1. Shuttle User App

    Used by passengers to:

  • Register / login
  • Search routes
  • Book shuttle rides
  • Track trips
    • 2. Shuttle Driver App

    Used by drivers to:

  • Accept ride requests
  • Manage trips
  • Update trip status
  • Receive ride notifications

    • Both apps must be added separately to Firebase.

    04 3. Import Project in Android Studio

    Step 1

    Open Android Studio

    Step 2

    Click:

    File → Open

    Step 3

    Select the project folder.

    Step 4

    Wait for Gradle Sync to complete.

    05 4. Change Application Name

    To change the App Name, update the `strings.xml` file.

    Navigate to:

    app/src/main/res/values/strings.xml

    Update the following section:

    Code 
    <!DOCTYPE resources [<!ENTITY appname "Shuttle Clone Captain">
<!ENTITY googleMapKey "API KEY">
<!ENTITY privacyPolicyUrl "https://shuttle.ridein.in/#/privacy-policy">
<!ENTITY termCondUrl "https://shuttle.ridein.in/#/terms">
<!ENTITY countryCode "+91">]>

<resources>
<string name="app_name" translatable="false">&appname;</string>

<string name="google_maps_key">&googleMapKey;</string>
    Update These Values

    appname → Your application name

    googleMapKey → Your Google Maps API Key

    privacyPolicyUrl → Your privacy policy URL

    termCondUrl → Your terms and conditions URL

    countryCode → Default country code

    06 5. Change Package Name (Recommended)

    If you plan to publish the app on Google Play, change the package name.

    Step 1

    Open:

    AndroidManifest.xml

    Step 2

    Change the package name.

    Code

    Step 3

    AppConstants.kt

    Example:

    • 19 15. Common Issues

    Gradle Sync Error

    Fix by restarting Android Studio:

    File → Invalidate Caches → Restart

    Google Maps Not Loading

    Check the following:

  • API key added correctly
  • Maps SDK for Android enabled
  • Internet permission enabled in AndroidManifest

    • OTP Not Working

    OTP is handled by the backend third-party OTP service.

    Make sure:

  • Backend API is running
  • BASE_URL is correct
  • Phone number API is reachable
